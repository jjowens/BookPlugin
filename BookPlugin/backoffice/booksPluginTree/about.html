Concepts

This is an example plugin demostrating the following concepts:

  1. Migration Plans - In your Umbraco database, it will create a new custom table named as "Book"" and then populate the table with new records. Those plans shows how it's done
  2. Retrieve Data - The controller in the "api" folder shows how it uses ScopeProvider to retrieve a records from your custom table, Book
  3. Tree Controllers - In the "trees" folder, the TreeController set up the menu items.

Reset Plugin

Once a migration plan has been executed, it cannot be re-run. However, if you need to reset it for experimental or development purposes, you can re-run the migration plans.

Please note don't do this on a production server. It could create a massive headache if something goes wrong.

To reset the migration plans, log into your Umbraco database. Look for the custom table "Book". Delete/drop the table "Book". If the table is not deleted, the migration plan will check if the table already exists

Then look for the "umbracoKeyValue" table. It holds key/value records. There should be a record with the key set to "Umbraco.Core.Upgrader.State+BooksPluginInit". The "BooksPluginInit" part in the key is the name of the migration plan.

Delete the record with the key set to "Umbraco.Core.Upgrader.State+BooksPluginInit". That's the reset completed.

Restart your Umbraco website, it will re-create the Book table
